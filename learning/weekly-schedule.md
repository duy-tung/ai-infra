# Weekly schedule — 8–10h/tuần

Bạn đi làm full-time → **đừng học 20h/tuần**. Duy trì **8–10h/tuần đều đặn** quan
trọng hơn học dồn. Đều > nhiều.

## Lịch tuần mẫu

| Ngày | Thời lượng | Việc |
|------|-----------|------|
| Thứ 2 | 1h | Đọc lesson/resource (Phase ưu tiên) |
| Thứ 3 | 1.5h | Code project |
| Thứ 4 | 1h | English technical writing (README/note/blog) |
| Thứ 5 | 1.5h | Code project |
| Thứ 6 | 0.5h | Ghi brag doc / notes |
| Thứ 7 | 3h | Build sâu (block lớn nhất) |
| Chủ nhật | 1h | Review tuần + plan tuần sau (cập nhật tracker) |

**Tổng: ~8.5h/tuần.** Điều chỉnh theo thực tế, nhưng giữ tỉ lệ:

> **70% build · 20% đọc · 10% viết/public.**

## Rule quan trọng

- Nếu chỉ **đọc course** → giá trị career thấp.
- Nếu **build artifact + viết lại bằng tiếng Anh** → giá trị cao hơn nhiều.
- Mỗi tuần kết thúc phải có **1 commit** vào repo này (code hoặc note).
- Mỗi Chủ nhật: cập nhật [`progress-tracker.md`](progress-tracker.md).

## Chu kỳ sprint

- Mỗi sprint ~3–4 tuần (xem [`sprints/`](sprints/)).
- Cuối sprint: 1 artifact + 1 bài viết ngắn tiếng Anh.

## Khi bận / nghỉ

Tuần bận: tối thiểu giữ **Thứ 7 (3h build)** + **Chủ nhật (1h review)**. Đừng để
đứt mạch quá 1 tuần — đứt mạch là kẻ thù số 1 của lộ trình dài.
