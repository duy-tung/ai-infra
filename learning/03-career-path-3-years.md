# 03 — Career path 3 năm

## Năm 1 — AI-native backend/platform owner trong fintech

**Mục tiêu:** thành người trong team hiểu nhất về **safe AI-assisted delivery.**

Cần đạt:
- Own một system critical: payment / ledger / reconciliation / risk / audit /
  transaction pipeline.
- Có AI PR review hoặc incident triage workflow (từ portfolio projects).
- Có **số liệu impact**.
- Có 1–2 design docs tiếng Anh.
- Có portfolio public/anonymized.

**CV bullet mẫu sau năm 1:**
> Built an AI-assisted review and verification workflow for fintech backend
> services, detecting idempotency, migration, audit, and transaction-boundary
> risks before merge; instrumented agent runs with OpenTelemetry, eval fixtures,
> and human approval gates.

## Năm 2 — Platform scope hoặc nhảy công ty

**Mục tiêu:** từ backend contributor → platform/system owner.

Nếu công ty hiện tại cho scope, xin own: dev productivity platform · agent
tooling · observability · CI/CD quality gates · incident automation · eval
framework · fintech reliability.

Nếu không cho scope → canh nhảy: ZaloPay (platform/reconciliation/payment) ·
MoMo (risk/payment/data/AI platform) · Techcombank/TCBS (data/platform/digital
banking) · Airwallex · GrabFin/Wise/Stripe.

> **Tiêu chí nhảy:** role mới phải cho **platform ownership**, không chỉ backend feature.

## Năm 3 — Apply global AI infra/platform

**Mục tiêu:** apply được AI Platform / Agent Infrastructure / Evals /
Observability / Developer Productivity.

Target roles:
- **OpenAI:** Agent Infrastructure · Evals · Observability · Developer
  Productivity · Core Services · Database Systems · Codex Core Agents.
- **Anthropic:** Research Tools · AI Observability · Safeguards Infrastructure ·
  Inference Systems.
- **Exa:** Backend/Infrastructure/Retrieval.
- **Airwallex/Stripe/Wise:** payment platform + AI/data infra.
- **Databricks/AWS/Azure AI:** platform/runtime/observability/model serving.

## Việc cần làm xuyên suốt (mọi năm)

- **Brag doc** ([`brag-doc.md`](brag-doc.md)): ghi mọi impact ngay khi xảy ra.
- **English technical writing**: README / design doc / blog — đều đặn (10% thời gian).
- **Portfolio public**: 3 repo trong `projects/` luôn có README, diagram, demo,
  "what I learned", "production considerations", "failure modes", "security
  model", "evaluation methodology".
